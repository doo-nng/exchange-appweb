# 환율 뉴스(상황 파악) 섹션 설계 — Phase 2

**날짜:** 2026-06-18
**목적:** 환율의 변동·이슈·추세를 빠르게 파악 (기사 읽기 X, 의미 정리 O). 그래프가 '가격 추세'를 받치므로, 이 섹션은 '드라이버와 이슈'를 담당.
**제약:** 비용 0원(무조건). PC·모바일 UI 단순. 개인용 단일 사용자.

---

## 1. 정보 모델 — A+B (브리핑 + 이슈 테마, 연속성 추적)

**① 오늘의 브리핑** (매일 새로 생성 — 스냅샷)
- `headline`: 오늘 한 줄
- `drivers`: 통화별 1줄 (예: "달러 ▲0.3% · 연준 매파 발언"), 움직임 큰 것 위주 3~5줄
- 입력: 당일 환율 변동치(앱 데이터) + 뉴스

**② 이슈 테마** (연속성 추적 — 어제 상태 반영)
- 고정 5테마: 미 연준·금리 / 한국경제·한은 / 중국 경기 / 일본·BOJ / 원자재·신흥국
- 각: `status`(현황 1줄) + `trend`(지속/심화/완화/진정)
- 매일 cron이 **어제 테마 상태를 AI에 넘겨** 갱신 → 긴 히스토리 없이 추적감

---

## 2. 수집 / 관리 / 노출

- **수집**: 네이버 검색 API(환율·달러·금리·연준·한은 등 쿼리) + 한국경제 RSS + ForexLive·FXStreet RSS + 당일 환율 변동치(Yahoo, 앱과 동일 소스). 키워드 필터 + 중복 제거 → 상위 ~25건.
- **관리**: Upstash `news:latest`에 **최신 브리핑+테마+출처만** 저장(누적 X). 매일 직전 값을 읽어 테마 연속성에 사용.
- **노출**: 그래프 아래 하단 섹션. 브리핑 블록 + 테마 칩 + 출처 링크. 탭 전환 없음.

---

## 3. 큐레이션 (Gemini 무료)

- 엔진: Google Gemini 무료 티어(Flash). 하루 1회 호출 → 무료 한도 내 0원.
- 프롬프트 입력: 당일 환율 변동 + 어제 테마 + 오늘 헤드라인(번호+출처). 출력: JSON(brief, themes). **URL·없는 사실 생성 금지.**
- 출처/링크는 **실제 수집한 피드 아이템**에서 직접(상위 ~5건) → AI URL 환각 방지.
- 실패/빈 결과 시 직전 `news:latest` 유지(덮어쓰지 않음).

---

## 4. 데이터 구조 (Upstash `news:latest`)

```json
{
  "updatedAt": "2026-06-18T08:00:00+09:00",
  "brief": { "headline": "…", "drivers": [ {"code":"USD","line":"달러 ▲0.3% · 연준 발언"} ] },
  "themes": [ {"title":"미 연준·금리","status":"매파 기조 지속","trend":"심화"} ],
  "sources": [ {"title":"…","source":"한국경제","url":"https://…"} ]
}
```

## 5. 파일 / 인프라

| 파일 | 내용 |
|---|---|
| `scripts/fetch-news.mjs` | 신규 — 수집·필터·Gemini·저장 (GitHub Actions 실행) |
| `.github/workflows/news.yml` | 신규 — 하루 1회 cron (08:00 KST = 23:00 UTC) + 수동 실행 |
| `api/get-news.js` | 신규 — Upstash `news:latest` 반환 |
| `lib/redis.js` | 수정 — 범용 redisGet/redisSet 추가 |
| `index.html` | 수정 — 하단 뉴스 섹션 + CSS + loadNews() |

### 시크릿 (GitHub Actions)
- 기존: `UPSTASH_REDIS_REST_URL/TOKEN`
- 신규(무료): `GEMINI_API_KEY`, `NAVER_CLIENT_ID`, `NAVER_CLIENT_SECRET`

### 비용
0원 — 무료 RSS + 네이버 무료 티어(2.5만/일) + Gemini 무료 티어 + 기존 인프라.
