**날짜:** 2026-05-19  
**범위:** index.html 단일 파일 수정

## 1. 환율 카드 — 배지 2개 표시

### 현재 상태
- 우측 상단 작은 배지 1개: `▲ 3.50 (+0.25%)` — 레이블 없어 무엇 대비인지 불명확
- 레인지 코멘트: "최근 1년 평균보다 42원 저렴한 구간이에요." (문장 형태)

### 변경 사항
큰 환율(`card-rate`) 바로 아래에 배지 2개를 나란히 배치:

```
전일 대비     ▲ +0.25%   |   1년 평균 대비   ▼ 42원 저렴
```

- **전일 대비 배지**: API 응답 `prevClose` 대비 변화율(%). 상승=빨강, 하락=파랑
- **1년 평균 대비 배지**: `historicalData` 평균 대비 현재가 차이(원). 저렴=파랑, 비쌈=빨강. JPY는 소수점 3자리, 나머지는 2자리.
- 각 배지 위에 작은 레이블(`전일 대비`, `1년 평균 대비`) 표시
- 기존 우측 상단 `change-badge` 제거 (중복)
- 기존 `range-comment` 문장 제거 (배지로 대체)
- 레인지 바(52주 레인지)는 그대로 유지

### 수용 기준
- 1년 데이터 부족(30개 미만)이면 `1년 평균 대비` 배지 숨김
- 카드 로딩 중(skeleton 상태)에는 배지 영역도 skeleton으로 표시
- **전일 대비 배지**는 변화율(%)만 표시. 차액(원) 표시 없음. JPY 포함 모두 동일.
- **1년 평균 대비 배지**의 차이값은 KRW 단위(원). USD/CNY: 소수점 2자리, JPY: 소수점 3자리

## 2. 차트 — 점 제거 + 툴팁 개선

### 현재 버그
- `pointRadius: 0` 설정에도 불구하고 점이 보임 (Chart.js 렌더링 이슈)
- 커서를 올리면 점이 사라지는 역방향 동작
- 툴팁에 등락률(%)만 표시, 실제 환율(원) 없음

### 변경 사항
- `pointRadius: 0`, `pointHoverRadius: 5`, `pointBorderWidth: 0` 명확히 설정
- 각 dataset 객체에 `baseRate` 프로퍼티 추가: 기간 시작 시점의 실제 환율(`filtered[0].close`)
- 툴팁 `label` 콜백 수정: `item.dataset.baseRate`와 `item.parsed.y`(등락률%)로 실제 환율 역산 후 함께 표시
  - 역산 공식: `actualRate = baseRate * (1 + pct / 100)`
  - 표시 형태: `USD: +2.5% · 1,382원`
  - CNY는 소수점 2자리, JPY는 3자리
- 툴팁 `title` 콜백은 기존 날짜 표시 유지

### 수용 기준
- 평상시 선에 점 없음
- 커서 올리면 해당 데이터포인트에 점 표시
- 툴팁: 날짜(title) + 통화별 `등락률% · 실제환율원`(label)

## 3. 금액 계산기 — 플로팅 팝오버

### 전제 조건 (코드 변경)
- 모듈 스코프에 `let currentRates = {}` 선언 추가
- `loadCurrentRates()` 완료 시 `currentRates` 갱신: `currentRates[d.code] = d.rate`
- 모듈 스코프에 `let isCalcOpen = false` 선언 추가 (팝오버 열림 상태 추적)
- 이후 계산기가 `currentRates`와 `isCalcOpen`을 참조

### 구조

**플로팅 버튼** (`#calc-fab`)
- `position: fixed; bottom: 24px; right: 20px; z-index: 200`
- 원형(48px), 파란 배경(`--accent`), 💱 아이콘
- 클릭 시 팝오버 토글
- 팝오버 열려있을 때 ✕ 아이콘으로 전환

**팝오버 카드** (`#calc-popover`)
- `position: fixed; bottom: 80px; right: 20px; z-index: 201`
- 너비: 300px, `max-width: calc(100vw - 40px)`
- 외부 클릭 시 닫힘: `document` click 핸들러에서 `event.target.closest('#calc-fab, #calc-popover')` 가 null이면 닫힘
- 토스트(`z-index: 999`) 아래 유지

**팝오버 내부 구성**
```
[ USD ] [ JPY ] [ CNY ]          ← 통화 탭

  원화 (KRW)
  [ 1,000,000        원 ]

        ⇄ (스왑 버튼)

  달러 (USD)
  [ 724.64          달러 ]
```

### 동작 규칙

**⇄ 스왑 버튼**: 입력 방향을 전환. 필드 위치는 유지하되 어떤 필드가 "입력" 역할인지 토글.
- 스왑 전: 원화 필드 입력 → 외화 필드 자동 갱신
- 스왑 후: 외화 필드 입력 → 원화 필드 자동 갱신
- 스왑 시 포커스가 현재 입력 필드로 이동

**계산 규칙**
- 탭 전환 시 현재 입력값 기준으로 즉시 재계산
- JPY: 내부 계산 시 `rate / 100` 적용 (100엔 기준 고시를 1엔 단위로 변환), UI 표시는 1엔 기준
- 환율 미로드(`currentRates[code]` 없음)이면 필드 대신 "환율을 불러오는 중입니다" 안내 표시
- `refreshAll()` 완료 후 `isCalcOpen === true`이면 자동 재계산 (`currentRates` 갱신 후 호출)

**입력 포맷**
- 숫자 및 소수점(`.`)만 허용, 다른 문자 무시
- 원화 입력: 정수 콤마 포맷(`1,000,000`)
- 외화 출력: 통화별 소수점 자리수 (USD/CNY: 2자리, JPY: 0자리 — 카드의 3자리와 다름, 계산기는 정수 엔으로 표시하는 것이 의도)

### 수용 기준
- 모바일(375px)에서 팝오버가 화면 밖으로 나가지 않음
- 환율 새로고침 후 팝오버 열려있으면 자동 재계산
- 1년 평균 대비 데이터 없어도 계산기는 정상 동작
