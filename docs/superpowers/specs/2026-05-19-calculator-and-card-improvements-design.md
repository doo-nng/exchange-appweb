**날짜:** 2026-05-19  
**범위:** index.html 단일 파일 수정

## 1. 환율 카드 — 배지 2개 표시

### 현재 상태
- 우측 상단 작은 배지 1개: `▲ 3.50 (+0.25%)` — 레이블 없어 무엇 대비인지 불명확
- 레인지 코멘트: "최근 1년 평균보다 42원 저렴한 구간이에요." (문장 형태)

### 변경 사항
큰 환율(`card-rate`) 바로 아래에 배지 2개를 나란히 배치:

```
전일 대비     ▲ +0.25%   |   1년 평균 대비   ▼ 42원 저렴
```

- **전일 대비 배지**: `chartPreviousClose` 대비 변화율(%). 상승=빨강, 하락=파랑
- **1년 평균 대비 배지**: `historicalData` 평균 대비 현재가 차이(원). 저렴=파랑, 비쌈=빨강
- 각 배지 위에 작은 레이블(`전일 대비`, `1년 평균 대비`) 표시
- 기존 우측 상단 `change-badge` 제거 (중복)
- 기존 `range-comment` 문장 제거 (배지로 대체)
- 레인지 바(52주 레인지)는 그대로 유지

### 수용 기준
- 1년 데이터 부족(30개 미만)이면 `1년 평균 대비` 배지 숨김
- JPY는 소수점 3자리 유지

## 2. 차트 — 점 제거 + 툴팁 개선

### 현재 버그
- `pointRadius: 0` 설정에도 불구하고 점이 보임 (Chart.js 렌더링 이슈)
- 커서를 올리면 점이 사라지는 역방향 동작
- 툴팁에 등락률(%)만 표시, 실제 환율(원) 없음

### 변경 사항
- `pointRadius: 0`, `pointHoverRadius: 5` 명확히 설정
- `pointBorderWidth: 0` 추가로 잔여 테두리 제거
- 툴팁 `label` 콜백 수정: 등락률(%) + 실제 환율(원) 함께 표시
  - 표시 형태: `USD: +2.5% · 1,382원`
  - 실제 환율은 기간 시작 기준가와 등락률로 역산
  - CNY는 소수점 2자리, JPY는 3자리

### 수용 기준
- 평상시 선에 점 없음
- 커서 올리면 해당 데이터포인트에 점 표시
- 툴팁에 날짜 + 통화별 등락률 + 실제 환율 표시

## 3. 금액 계산기 — 플로팅 팝오버

### 구조

**플로팅 버튼** (`#calc-fab`)
- `position: fixed; bottom: 24px; right: 20px; z-index: 100`
- 원형(48px), 파란 배경(`--accent`), 💱 아이콘
- 클릭 시 팝오버 토글
- 팝오버 열려있을 때 버튼 색 반전(X 아이콘)

**팝오버 카드** (`#calc-popover`)
- 버튼 바로 위, 오른쪽 정렬
- `position: fixed; bottom: 80px; right: 20px`
- 너비: 300px (모바일 `max-width: calc(100vw - 40px)`)
- 외부 클릭 시 닫힘

**팝오버 내부 구성**
```
[ USD ] [ JPY ] [ CNY ]          ← 통화 탭

  원화 (KRW)
  [ 1,000,000        원 ]

        ⇄ (스왑 버튼)

  달러 (USD)
  [ 724.64          달러 ]
```

### 동작 규칙
- 탭 전환 시 현재 입력값 기준으로 즉시 재계산
- 원화 필드 입력 → 외화 필드 실시간 갱신
- 외화 필드 입력 → 원화 필드 실시간 갱신
- ⇄ 버튼: 두 필드 입력 방향 스왑 (포커스 이동)
- 환율 소스: 로드된 `currentRates` 재사용, 추가 API 호출 없음
- 환율 미로드 상태면 "환율을 불러오는 중입니다" 안내
- JPY: 계산 시 100엔 단위 변환 내부 처리, 표시는 1엔 기준

### 수용 기준
- 모바일(375px)에서 팝오버가 화면 밖으로 나가지 않음
- 숫자 외 입력 무시(콤마 자동 포맷)
- 환율 새로고침 후 계산기 열려있으면 자동 재계산
