## 1. 범위 및 목적

이 문서는 환율 트렌드 대시보드 웹앱(Exchange Rate Dashboard PWA)의 기능 요구사항, 시스템 동작 명세, 데이터 계약, 에러 처리 정책을 정의한다.

**다루는 것:** 환율 현황 표시, 트렌드 그래프, 환전 적기 판단 UI, 데이터 업데이트 정책, 에러/폴백 상태, PWA 설치 지원.

**다루지 않는 것:** 환율 알림, 금액 계산기, AI 인사이트, 뉴스 피드, 회원 인증. (서비스기획서 NICE TO HAVE / OUT OF SCOPE 항목)

## 3. 기능 요구사항 (FR)

### FR-001 — 환율 카드 표시

**요구사항:** 달러(USD), 엔화(JPY), 위안화(CNY) 각각에 대해 독립된 카드를 렌더링한다.

**수용 기준:**
- 각 카드는 통화명, 현재 시장 환율(FX), 전일 대비 변화량을 표시한다.
- 전일 대비 변화(절댓값 + 방향 화살표)를 표시한다. 상승은 적색, 하락은 청색, 변동 없으면 회색.
- 데이터 출처를 카드 하단에 소문자로 표기한다: "Yahoo Finance · FX 시장 기준".
- 값이 null이면 해당 셀에 `--`를 표시한다.
- 카드 하단에 "실제 환전 시 은행 스프레드 별도 적용" 안내 문구 표시.

**JPY 특이사항:**
- Yahoo Finance `JPYKRW=X`는 **1엔당 KRW** 반환 (예: 9.41원/엔).
- 카드에는 "1엔 기준"으로 표시한다.

**엣지 케이스:**
- 전일 종가 데이터가 없어 변화량 계산 불가 시: 변화량 필드를 숨기고 전일 비교 없이 현재값만 표시.

---

### FR-002 — 데이터 자동 로드 & 수동 새로고침

**요구사항:** 앱 진입 시 자동으로 최신 환율 데이터를 가져오고, 사용자가 새로고침 버튼을 눌러도 재요청한다.

**수용 기준:**
- 앱 진입(초기 로드 / PWA 포그라운드 복귀) 시 자동 fetch 실행.
- 새로고침 버튼 탭 시: 버튼에 로딩 스피너 표시 → 요청 완료 후 카드 값 및 타임스탬프 갱신.
- 요청 중 이전 값은 그대로 유지(빈 화면 플래시 없음).

**엣지 케이스:**
- 요청 실패 시: 이전 값을 유지하고, 상단 토스트로 "업데이트 실패 · 이전 데이터를 표시합니다" 안내. 새로고침 버튼은 재활성화.

---

### FR-003 — 마지막 업데이트 시각 표시

**요구사항:** 현재 환율 데이터의 고시 시각을 항상 화면에 표시한다.

**수용 기준:**
- 포맷: `수출입은행 기준 · MM월 DD일 HH:MM 업데이트`
- 앱 진입 시 데이터 fetch 완료 후 즉시 갱신.
- 영업일 오전 11시 이전에 앱을 열 경우: 고시 시각 옆에 `(전일 기준)` 레이블 추가 표시.
- 데이터를 한 번도 불러오지 못한 상태: `업데이트 정보 없음`.

---

### FR-004 — 11시 이전 폴백 안내

**요구사항:** 영업일 오전 11시 이전 앱 진입 시 당일 환율이 미고시 상태임을 사용자에게 알린다.

**수용 기준:**
- 현재 시각이 영업일 오전 11:00 이전이고, API가 전일 데이터를 반환하는 경우:
  - 카드 상단에 배너 표시: `"오늘 환율은 오전 11시 이후 업데이트됩니다 · 전일 기준으로 표시 중"`
  - FR-003의 타임스탬프에 `(전일 기준)` 레이블 추가.
- 주말·공휴일에는 최근 영업일 데이터를 표시하고 동일 배너 노출.

**판단 로직:**
```
현재 시각 < 11:00 KST → 전일 기준 배너 노출
현재 시각 ≥ 11:00 KST → 당일 기준. 배너 없음.
```

### FR-005 — 52주 레인지 Progress Bar

**요구사항:** 통화별로 현재 환율이 최근 52주(1년) 최저·최고 범위에서 어느 위치인지 Progress bar로 시각화한다.

**수용 기준:**
- 각 통화 카드 하단 또는 별도 섹션에 Progress bar 표시.
- Progress bar 위치 계산:
  ```
  position = (현재가 - 52주 최저) / (52주 최고 - 52주 최저) × 100 (%)
  ```
- 양 끝에 52주 최저값, 52주 최고값 레이블 표시.
- **색상 규칙:**
  - 현재가 < 1년 평균 → 파란색 (저렴 구간)
  - 현재가 ≥ 1년 평균 → 빨간색 (비싼 구간)
- Progress bar 아래 한 줄 문장:
  - 저렴 구간: `"현재 [통화명]은 최근 1년 평균([평균값]원)보다 [차이]원 저렴한 구간이에요."`
  - 비싼 구간: `"현재 [통화명]은 최근 1년 평균([평균값]원)보다 [차이]원 비싼 구간이에요."`

**데이터 소스:** Frankfurter API (ECB 기준 과거 1년 일별 데이터). 출처 표기: "ECB 기준 과거 1년".

**엣지 케이스:**
- 과거 데이터 fetch 실패 시: Progress bar 미표시, "레인지 정보를 불러오지 못했습니다" 텍스트.
- 52주 최저 = 52주 최고인 경우(사실상 불가능하나 방어): bar를 50% 위치에 고정.

### FR-006 — 환율 트렌드 그래프

**요구사항:** USD, JPY, CNY의 최근 1년 환율 변화를 라인 차트로 표시한다.

**수용 기준:**
- 기간 탭: `1W` / `1M` / `3M` / `6M` / `1Y` (기본값: 1M)
- 탭 선택 시 해당 기간 데이터로 차트 즉시 업데이트 (re-fetch 없이 클라이언트 필터링).
- 차트에는 날짜(X축)와 환율(Y축) 표시. Y축 단위: 원(KRW).
- JPY: Y축 단위를 "100엔당 원"으로 표기.
- 데이터 포인트 마우스오버/탭 시 날짜 + 환율 툴팁 표시.
- 차트 하단에 데이터 출처 표기: "ECB(Frankfurter) 기준".

**데이터 소스:** Frankfurter API. 기간: 현재일 기준 -365일 ~ 현재일.

**엣지 케이스:**
- 데이터 로딩 중: 차트 영역에 스켈레톤 UI.
- 데이터 fetch 실패: "그래프를 불러오지 못했습니다. 새로고침 해주세요." 인라인 메시지.
- 선택 기간 내 데이터 0건: "해당 기간 데이터가 없습니다."

---

### FR-007 — PWA 홈 화면 설치 지원

**요구사항:** 아이폰 사파리에서 홈 화면 추가(Add to Home Screen) 시 네이티브 앱처럼 실행 가능해야 한다.

**수용 기준:**
- `manifest.json` 포함. 필수 필드: `name`, `short_name`, `icons` (192x192, 512x512), `start_url`, `display: standalone`, `background_color`, `theme_color`.
- 홈 화면 아이콘에서 실행 시 주소 표시줄 없이 전체 화면으로 열림.
- iOS Safari에서 홈 화면 추가 가능 (Safari 공유 메뉴 → "홈 화면에 추가").

---

### FR-008 — 반응형 레이아웃

**요구사항:** 맥북 브라우저(1280px 이상)와 아이폰 화면(390px 이하) 양쪽에서 정상 사용 가능해야 한다.

**수용 기준:**
- 모바일(< 768px): 카드 1열 세로 스택. 그래프 전체 너비.
- 데스크탑(≥ 768px): 카드 3열 또는 2열 그리드 허용. 그래프 좌측 정렬 or 전체 너비.
- 텍스트 잘림(truncation) 없이 모든 레이블이 표시되어야 한다.
- 터치 타겟 최소 44×44px (iOS HIG 기준).

## 5. 데이터 요구사항

### 5.1 대상 통화 및 데이터 모델

```
CurrencyRate {
  code: string          // "USD" | "JPY" | "CNY"
  name: string          // "미국 달러" | "일본 엔화" | "중국 위안화"
  baseRate: number | null    // 매매기준율 (KRW 기준)
  buyRate: number | null     // 현찰 살 때
  sellRate: number | null    // 현찰 팔 때
  changeFromPrev: number | null  // 전일 대비 (+ 상승 / - 하락)
  unit: number           // USD·CNY: 1 / JPY: 100
  source: "수출입은행"
  updatedAt: ISO8601 string
}

HistoricalRate {
  date: string          // "YYYY-MM-DD"
  rate: number          // 해당 통화의 KRW 환산 환율 (JPY: 100엔 기준)
  source: "ECB"
}
```

### 5.2 외부 데이터 소스 계약

**한국수출입은행 Open API**

| 항목 | 내용 |
|------|------|
| 제공 데이터 | 기준환율, 살 때, 팔 때 (영업일 1일 1회 고시) |
| 업데이트 시점 | 영업일 오전 11시 이후 |
| 보장 필드 | `cur_nm`, `deal_bas_r`, `ttb`, `tts` |
| Null 가능 | 영업일 아닌 날, 11시 이전 |
| JPY 단위 | 100엔 기준 고시 |
| 호출 한도 | 일 1,000회 |

**Frankfurter API (ECB 기준)**

| 항목 | 내용 |
|------|------|
| 제공 데이터 | ECB 기준 일별 환율 (1999년~현재) |
| 업데이트 시점 | 영업일 기준 1일 1회 |
| 범위 조회 | `GET /YYYY-MM-DD..YYYY-MM-DD?to=KRW&from=USD,JPY,CNY` |
| 보장 필드 | `rates[date][currency]` |
| Null 가능 | 주말, 공휴일 (해당 날짜 키 없음) |
| 호출 한도 | 없음 (오픈소스 공개 API) |
| 주의 | ECB 기준이라 수출입은행 값과 수치 차이 있음 |

### 5.3 업데이트 정책

| 트리거 | 동작 |
|--------|------|
| 앱 진입 (초기 로드) | 수출입은행 API fetch (현재 환율) |
| 앱 진입 (최초 1회) | Frankfurter API fetch (과거 1년 데이터) |
| 새로고침 버튼 탭 | 수출입은행 API 재호출 |
| Frankfurter 데이터 캐시 만료 | 하루 1회 이상 요청하지 않음 (같은 날 재요청 스킵) |
